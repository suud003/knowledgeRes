# /track <plan_id> <week>

## 功能
记录 Ship-Learn-Next 学习计划的周迭代执行结果

## 参数
- `plan_id`（必填）：计划 ID 或简写名称，如 "Prompt工程"
- `week`（必填）：周次（1-4）

## 执行步骤

1. **引导记录**：依次询问用户以下内容（如果用户未在一次性消息中提供）：
   - 🚀 **SHIP（交付）**：这周实际完成了什么？
   - 🧠 **LEARN（反思）**：什么有效？什么无效？有什么新发现？
   - ➡️ **NEXT（调整）**：下周计划做哪些调整？（可选）
2. **保存记录**：调用 `track_iteration` MCP 工具
3. **展示总结**：向用户展示记录结果和累计进度

## 输出格式示例

```
✅ Week 2 迭代记录已保存！

🚀 SHIP：完成了 3 个工作场景的 Prompt 优化
🧠 LEARN：简洁指令效果更好，复杂模板反而容易出错
➡️ NEXT：下周尝试将优化后的 Prompt 应用到实际项目中

📊 计划进度：██████░░░░ 50%（2/4 周）
```
