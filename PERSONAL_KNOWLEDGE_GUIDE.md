# Moos 个人知识库构建指南

## 🎯 核心价值

Moos 不是另一个笔记软件，而是**你和AI共同构建的知识基础设施**。通过MCP协议，AI可以直接操作你的知识库，实现真正的「人机协作」。

## 📋 快速开始清单

### 1. 验证MCP服务器配置

首先确认MCP服务器已正确配置：

```python
# 在Claude Code中测试以下命令
list_topics()  # 应该返回主题列表
```

如果返回类似这样的结果，说明配置成功：
```
['personal', 'writing', 'tasks', 'preferences', 'reflection', 'product', 'work', 'reading', 'ideas', 'ai']
```

## 📊 数据管理和备份

### 文件结构说明
```
data/
├── context/           # 知识库（Git备份）
│   ├── __created/     # 你的原创内容
│   └── __collected/    # 收集的内容
└── raw/               # 原始数据（Git忽略）
    └── generated/      # AI生成内容
```

### 备份策略
- **context目录**：使用Git进行版本控制
- **raw目录**：本地备份，不纳入版本控制
- **定期同步**：建议每周进行一次Git提交

## 🛠️ 故障排除

### 常见问题

1. **MCP工具无法调用**
   - 检查cwd路径配置是否正确
   - 确认项目目录有读写权限
   - 重启Claude Code

2. **内容分类不准确**
   - 调整config.yaml中的关键词
   - 手动纠正错误分类
   - 训练AI理解你的分类偏好

3. **存储空间不足**
   - 定期清理raw/generated目录
   - 使用Git进行版本控制，删除历史大文件
