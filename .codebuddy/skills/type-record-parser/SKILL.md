# 打字记录解析器（Type Record Parser）

> 将键盘输入法的打字记录还原为可读的中文文字，作为日报/周报生成的数据预处理环节。

## 概述

用户的打字数据来自键盘记录工具，格式为拼音输入法的操作序列。本 Skill 负责将这些操作序列还原为用户实际输入的文字内容。

## 数据格式说明

### 整体结构

每行格式为：
```
[HH:MM:SS] <操作序列>
```

时间戳 `[HH:MM:SS]` 标记该行输入发生的时间。

### 操作类型详解

#### 1. 拼音输入：`[拼音:xxx]`

表示用户通过拼音输入法输入了拼音 `xxx`。拼音后面必须跟一个选字操作。

**示例：**
- `[拼音:nihao]` → 用户输入了拼音 "nihao"

#### 2. 选字操作：`[选字:N]` / `[选字:回车]` / `[选字:空格]`

紧跟在 `[拼音:xxx]` 后面，表示用户从候选列表中选择了第 N 个词。

**选字规则：**

| 选字值 | 含义 | 转换规则 |
|--------|------|---------|
| `[选字:1]` | 选择第1候选词 | 将拼音转换为最常见的对应中文词 |
| `[选字:2]` | 选择第2候选词 | 将拼音转换为第2常见的对应中文词 |
| `[选字:3]` | 选择第3候选词 | 将拼音转换为第3常见的对应中文词 |
| `[选字:N]` | 选择第N候选词 | 将拼音转换为第N常见的对应中文词 |
| `[选字:回车]` | 英文直出 | **拼音原样保留为英文字母**，不做中文转换 |
| `[选字:空格]` | 确认第1候选词 | 等同于 `[选字:1]` |

**重要：`[选字:回车]` 是英文直出模式**，此时拼音不转换为中文，而是直接输出拼音字母本身。例如：
- `[拼音:game][选字:回车]` → 输出 "game"
- `[拼音:api][选字:回车]` → 输出 "api"
- `[拼音:bruce][选字:回车]` → 输出 "bruce"
- `[拼音:skill][选字:回车]` → 输出 "skill"

#### 3. 退格操作：`[退格]`

删除当前光标前的一个字符。连续多个 `[退格]` 表示连续删除多个字符。

**示例：**
- `你好[退格]` → 结果为 "你"（删除了"好"）
- `你好[退格][退格]` → 结果为 ""（删除了"好"和"你"）

#### 4. 回车操作：`[回车]`

表示用户按下了回车键，通常意味着一条消息发送完毕或换行。

**注意区分：**
- `[回车]` 单独出现 → 确认/发送消息
- `[选字:回车]` 在选字操作中 → 英文直出（不是换行）

#### 5. 直接字符输入

不在方括号内的字符，是用户直接输入的，原样保留。

**示例：**
- `123` → 直接输出 "123"
- `/` → 直接输出 "/"
- `9` → 直接输出 "9"

#### 6. Tab 键：`[Tab]`

表示 Tab 操作，通常可忽略或根据上下文处理。

## 拼音到中文的转换规则

### 核心原则

拼音输入法的候选词排列遵循"使用频率"规则：
- **选字:1**（或选字:空格）= 该拼音最常见、最高频的词
- **选字:2** = 第二常见的词
- **选字:N** = 第N常见的词

### 转换策略

由于我们无法精确复原输入法当时的候选词列表，需要采用以下策略：

#### 选字:1 或 选字:空格（最高频词）

直接将拼音转换为最常见的对应中文。利用以下优先级：

1. **整词匹配优先**：如果拼音是一个完整词的拼音，优先转为该词
   - `nihao` → "你好"
   - `xiexie` → "谢谢"
   - `gongzuo` → "工作"
   
2. **上下文推断**：结合前后文推断最可能的词
   - `[拼音:kaifa][选字:1]` 在技术语境中 → "开发"
   
3. **常见词优先**：选择日常使用频率最高的词
   - `shi` → "是"（而非"十"、"时"、"事"等）

#### 选字:2、3、... （非首选词）

需要根据拼音和上下文推断用户选择了哪个词。常见情况：

- `[拼音:zai][选字:2]` → "再"（首选通常是"在"）
- `[拼音:de][选字:1]` → "的"；`[拼音:de][选字:2]` 可能是 "得"
- `[拼音:ba][选字:2]` → "把"（首选可能是"吧"）
- `[拼音:le][选字:1]` → "了"；`[拼音:le][选字:2]` 可能是 "乐"
- `[拼音:huo][选字:1]` → "火"；`[拼音:huo][选字:3]` 可能是 "活" 或 "或"

**关键策略**：对于非首选词，必须结合前后文语义来推断最合理的词。不必强求100%精确还原，但要确保语义通顺。

#### 选字:回车（英文直出）

拼音字母原样输出，不做任何中文转换：
- `[拼音:vibe][选字:回车]` → "vibe"
- `[拼音:note][选字:回车]` → "note"
- `[拼音:cmd][选字:回车]` → "cmd"

### 拼音中的错别字和乱码处理

用户在快速打字时经常会出现拼音拼错的情况：
- `[拼音:djasnfjnsfgn]` → 明显是乱码，结合退格操作，这部分内容会被删除
- `[拼音:neirogn]` → 实际应为 "neirong"（内容），需智能纠错
- `[拼音:buabhu]` → 用户输错后又退格修正了

**处理规则：**
1. 如果拼音后紧跟大量退格，说明用户发现输错并删除了，可以忽略这段
2. 如果拼音略有拼写错误但可推断，尝试纠正（如 `neirogn` → `neirong` → "内容"）
3. 如果完全无法识别且没有退格修正，标记为 `[?]` 待确认

## 转换流程

### 步骤1：逐行解析

对每一行打字记录：
1. 提取时间戳
2. 按顺序解析操作序列
3. 维护一个"当前文本缓冲区"

### 步骤2：模拟输入过程

按操作顺序模拟：

```
缓冲区 = ""

遇到 [拼音:xxx][选字:N]:
  → 将xxx转换为对应中文词，追加到缓冲区

遇到 [拼音:xxx][选字:回车]:
  → 将xxx作为英文字母，追加到缓冲区

遇到 [退格]:
  → 从缓冲区末尾删除一个字符

遇到 [回车]:
  → 当前缓冲区内容作为一条完整消息输出
  → 清空缓冲区

遇到 直接字符:
  → 追加到缓冲区
```

### 步骤3：整理输出

将所有消息按时间排列，格式为：

```
[HH:MM] 还原后的中文文字
[HH:MM] 还原后的中文文字
...
```

### 步骤4：上下文修正

对初步还原结果进行整体检查：
- 修正明显不通顺的地方
- 结合上下文纠正可能的选字错误
- 合并同一时间段的碎片消息

## 完整示例

### 输入：
```
[10:26:04] [拼音:kaifa][选字:1][拼音:neibu][选字:1] [拼音:guanyu][选字:1][拼音:wow][选字:回车]
[10:26:17] [拼音:shiyong][选字:1]bankaifangpingt[拼音:bankaifang][选字:1][拼音:pingtai][选字:1][拼音:zuo][选字:1][拼音:wanfa][选字:1] [拼音:xuyao][选字:1][拼音:taolun][选字:1][回车]
```

### 解析过程：

**第1行 [10:26:04]：**
1. `[拼音:kaifa][选字:1]` → "开发"
2. `[拼音:neibu][选字:1]` → "内部"
3. ` ` → 空格
4. `[拼音:guanyu][选字:1]` → "关于"
5. `[拼音:wow][选字:回车]` → "wow"（英文直出）

缓冲区："开发内部 关于wow"
→ 没有回车，文本继续

**第2行 [10:26:17]：**
1. `[拼音:shiyong][选字:1]` → "使用"
2. `bankaifangpingt` → 直接字符 "bankaifangpingt"（用户可能在切换输入法前误输入）
3. `[拼音:bankaifang][选字:1]` → "半开放"
4. `[拼音:pingtai][选字:1]` → "平台"
5. `[拼音:zuo][选字:1]` → "做"
6. `[拼音:wanfa][选字:1]` → "玩法"
7. ` ` → 空格
8. `[拼音:xuyao][选字:1]` → "需要"
9. `[拼音:taolun][选字:1]` → "讨论"
10. `[回车]` → 消息结束

缓冲区涉及英文误输入，结合上下文整理。

### 输出：
```
[10:26] 开发内部 关于wow使用半开放平台做玩法 需要讨论
```

## 带退格的复杂示例

### 输入：
```
[10:30:35] [拼音:suir][选字:1][拼音:wojued][选字:1][拼音:meisha][选字:1][退格][退格][退格][退格][退格][退格][退格][退格][退格]
[10:30:42] [拼音:haishgi][选字:1][拼音:kehud][选字:1][拼音:d][选字:1][拼音:tongxue][选字:1][退格][退格][拼音:lingdaoduiwohao][选字:1][回车]
```

### 解析过程：

**第1行 [10:30:35]：**
1. `[拼音:suir][选字:1]` → "虽然"（2字）
2. `[拼音:wojued][选字:1]` → "我觉得"（3字）→ 注意拼音纠错 wojued → wojuede
3. `[拼音:meisha][选字:1]` → "没啥"（2字）
4. 9个退格 → 删除9个字符：从"虽然我觉得没啥"(7字) 删9个，但只有7字+可能其他字符

缓冲区在退格后为空或接近空

**第2行 [10:30:42]：**
1. `[拼音:haishgi][选字:1]` → "还是"（纠错：haishgi → haishi）
2. `[拼音:kehud][选字:1]` → "客户端"（纠错：kehud → kehuduan 的简写）
3. `[拼音:d][选字:1]` → "的"
4. `[拼音:tongxue][选字:1]` → "同学"
5. 2个退格 → 删除"同学"
6. `[拼音:lingdaoduiwohao][选字:1]` → "领导对我好"
7. `[回车]` → 消息结束

### 输出：
```
[10:30] 还是客户端的领导对我好
```

## 特殊情况处理

### 1. 长拼音（整句输入）

有些拼音很长，是用户一次性输入的整个短语：
- `[拼音:bankaifangpingtaideshixiang][选字:1]` → "半开放平台的事项"
- `[拼音:jinkenengduode][选字:1]` → "尽可能多的"
- `[拼音:yinqingjiekou][选字:1]` → "引擎接口"

### 2. 拼音片段被截断

用户输入到一半就退格重新输入：
- `[拼音:kaifan][选字:1][退格][退格][拼音:kaigang]` → 用户先输入"开饭"，觉得不对，退格后改为"开港"或其他

### 3. 混合输入

一行中同时包含中文拼音输入和英文直接输入：
- `[拼音:shiyong][选字:1]API` → "使用API"

### 4. 命令输入

以 `/` 开头的通常是命令，如 `/note`，保留原样。

### 5. 数字输入

直接数字保留原样：
- `7000` → "7000"
- `30` → "30"

## 与日报 Skill 的协作

本 Skill 的输出是 `daily-weekly-report` Skill 的输入：

```mermaid
graph LR
    A[键盘打字记录文件] -->|type-record-parser| B[可读中文文本]
    B -->|daily-weekly-report| C[日报/周报]
    C -->|iwiki-doc| D[发布到 iWiki]
```

**工作流：**
1. 用户提供打字记录文件（如 `keyboard_2026-03-05.txt`）
2. 本 Skill 将其转换为可读中文文本
3. 将转换结果传递给 `daily-weekly-report` Skill 进行工作内容识别和日报生成

## 使用方式

### 场景1：直接解析文件

```
用户：帮我解析今天的打字数据 [附带文件]
```

### 场景2：作为日报流程的一部分

```
用户：根据我的打字数据生成今天的日报
```

此时自动先调用本 Skill 解析打字数据，再调用 daily-weekly-report 生成日报。

### 场景3：仅转换不生成日报

```
用户：把这个打字记录转换成文字就行
```
